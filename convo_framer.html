Visualizing Chatbot Conversation:
Motion Design with Framer

Prototyping, User Research

Introduction

Since my team in Yahoo Search is interested in conversational design, I wanted to understand what it means to simulate a more "natural" interaction between machine and user. Here, I explored the concept of an agent typing back to a user. I'll provide very high level insight into my prototyping thought process. Unlike my prototyping for VUI (see my Alexa Skill making prototype), I mimic texting interactions using Framer, which is a tool I often use for native mobile experiences.


Framer Design Setup

I set up all of my screen artboards and layer groupings in the Design tab of Framer Studio, making sure to enable specific layers to be programmatically selectable for animating with later. I strategically make certain layer groupings have zero opacity or move off screen before sliding them in decoratively with code.



Example Typing Rhythm

The design tension here is to create a speed such that the subtle details of sliding and bouncing show; however, if the speed is too slow, it might be annoying for the user in terms of lag time and seeming too artificial.



Talking with the Bot

Here is a sample interaction where a user can tap on canned responses (bottom pills) to quickly answer a bot prompt. I am still figuring out the best speeds to account for longer text to animate in after the dots fade away.

Lessons Learned

  1. Users seemed to enjoy the typing simulations in lab studies. However, we would need to consider how the typing would affect user delight over time.
  2. Engineers had a tough time translating this to clean production code (we wanted to deploy to both Android and iOS), so I had to greatly simplify the animation subtleties.
