Auto-making Alexa Skills with
Google Spreadsheets and Node

Technical Prototyping

Introduction

As a prototyper concerned about bulk experimentation to uncover untapped opportunities, I did not see a sufficient VUI-based tool in the market that met my need. Instead, I saw this as an invitation to invent a tool that utilizes the Google Spreadsheets API to parse high volumes of utterances and expected responses to generate multiple Alexa Skill rapid prototypes. This tool is compatible with Amazon's Alexa Skills Kit CLI to create and deploy batches of Skills in under 5 minutes.

This tool was the cornerstone to my design efforts behind Crafting an NBA Q&A Experience through Alexa project, and currently only supports 1-Turn interactions. I also have been disseminating my methods within the community of practice I've started within the company, and am working towards scaling the tool visibility and scope to support broader problems.


New Prototyping Methodology

I wanted to invent a workflow that enabled the entire team to brainstorm and design answers in a common tool: Google Spreadsheets seemed to be the best for parallel editing and visualizing text comparisons. I also wanted to create an automation that stayed true to what is supported by the Alexa SDK in the case of needing to evolve the Skill prototypes themselves.

My script will parse a Google Spreadsheet and map one utterance (e.g. a user's question) with N possible answer variations. Each variation column is eventually translated into an Alexa Skill, where the intent, utterance, and answer rows represent a possible turn.

Using ASK CLI for Deploying

After overriding the default files in an ASK CLI template, we can deploy a Skill in a jiffy! The work for the developer (and fellow stakeholders) is more in the planning and designing of the interactions!



Testing on Echosim

Aside from the CLI and my Echo devices, I'm using Echosim.io for simulation. After hooking up my Amazon account, I can easily access my Alexa Skill locally even if I haven't deployed to the Alexa Skill Store.

Lessons Learned

  1. This was a super effective method of rapid prototyping for basic VUI. It allowed the team to test all of our flavors of utterances with little effort, compared to the manual and more cumbersome prototyping tools currently in existence. My larger vision is to support multimodality and multi-turn use cases.
  2. Given that my team of knowledge engineers have yet to delve into Alexa Skill making, I was able to provide early insight about the Alexa NLU and the technology suite in general. Essentially, it kept us grounded with what's truly feasible.
  3. I imagine that this is the beginning of streamlining workflows and enabling powerful expression for a designer. This could be a helpful tool for non-programmers to map out their Skill design in Google Spreadsheets and interact with developers with more ease.
